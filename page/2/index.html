远程分支

远程分支位于本地仓库中，反映了最后一次与远程仓库通信时的状态。

远程分支的命名规范是<remote name>/<branch name>，默认的远程仓库名是origin.

远程分支在成为checkout的目标时会自动进入分离HEAD状态，这是为了让远程分支不能成为直接操作的对象。

git clone时会自动为远程仓库中的每一个分支，在本地仓库中创建一个远程分支并加以关联。
